# Ancestree — Backend API

REST API for the [Ancestree](https://knownyourancestry.com) family graph platform.

| Environment | URL |
|---|---|
| Production | `https://api.knownyourancestry.com` |
| Dev | `https://dev.api.knownyourancestry.com` |

## Stack

| Layer | Tech |
|---|---|
| Runtime | Node.js 22 |
| Framework | Express 5 |
| Language | TypeScript → compiled to `dist/` |
| Database | PostgreSQL via **Neon** (connection pooling) |
| Auth | JWT (`jsonwebtoken`) |
| Validation | Zod |
| Logging | pino + pino-http |
| Rate limiting | express-rate-limit |
| Migrations | Custom runner (`database/migrate.ts`) — runs on every boot |
| Deploy | **Render** (free web service, `render.yaml`) |

## Local development

```bash
npm install
cp .env.example .env          # then edit the values
npm run dev                   # ts-node src/index.ts → http://localhost:4000
```

Health check: `GET http://localhost:4000/health`

### `.env` for local dev

```env
DATABASE_URL=postgresql://postgres:yourpassword@localhost:5432/ancestree
PORT=4000
NODE_ENV=development
LOG_LEVEL=debug
JWT_SECRET=any-local-secret
PLATFORM_ADMIN_KEY=any-local-key
FRONTEND_URL=http://localhost:3000,http://localhost:5173
```

Leave `DATABASE_SSL` unset for a local Postgres instance.

## Build

```bash
npm run build    # tsc → dist/  +  copies SQL migrations to dist/database/migrations/
npm start        # node dist/src/index.js
```

Migrations run automatically on every boot — no manual step needed.

## Deploy to Render

### Production (branch: `main`)

1. Render → **New → Blueprint** → connect `Dipu2552003/Ancestree-backend`, branch `main`
2. Render reads `render.yaml` and pre-fills most env vars
3. Set the two `sync: false` vars in the dashboard:
   - `DATABASE_URL` → Neon **production** branch **pooled** connection string
   - `FRONTEND_URL` → `https://knownyourancestry.com,https://3d.knownyourancestry.com`
4. `JWT_SECRET` and `PLATFORM_ADMIN_KEY` are auto-generated by Render
5. Deploy → watch logs for `✓ Migrations complete` then `server started`
6. Settings → Custom Domains → add `api.knownyourancestry.com`

### Dev (branch: `dev`)

1. Render → **New → Web Service** → same repo, branch **`dev`**, name it `ancestree-api-dev`
2. Build command: `npm ci --include=dev && npm run build`
3. Start command: `npm start`
4. Health check path: `/health`
5. Set env vars:

| Variable | Value |
|---|---|
| `DATABASE_URL` | Neon **develop** branch pooled URL |
| `DATABASE_SSL` | `require` |
| `NODE_ENV` | `production` |
| `LOG_LEVEL` | `debug` |
| `JWT_SECRET` | New random secret — **distinct from prod** |
| `PLATFORM_ADMIN_KEY` | New random secret — **distinct from prod** |
| `FRONTEND_URL` | `https://dev.knownyourancestry.com,https://3d.knownyourancestry.com` |

6. Settings → Custom Domains → add `dev.api.knownyourancestry.com`

### Keep-warm (production only)

Free Render services sleep after ~15 min idle (~50s cold start on wake).
Point **UptimeRobot** (free) at `https://api.knownyourancestry.com/health` every **10 min**.
Do **not** ping the dev service — let it sleep to stay within the 750 free instance-hours/month.

## Branch → environment mapping

| Branch | Deploys to | Neon branch |
|---|---|---|
| `main` | `api.knownyourancestry.com` | `production` |
| `dev` | `dev.api.knownyourancestry.com` | `develop` |

Workflow: feature branches off `dev` → merge to `dev` (test on dev URL) → merge to `main` (auto-deploys to prod).
